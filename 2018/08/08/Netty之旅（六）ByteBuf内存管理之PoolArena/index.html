Netty之旅（六）ByteBuf内存管理之PoolArena

发表于 2018-08-08 | 分类于 Netty之旅 |

  Netty之旅（五）ByteBuf内存管理之PoolSubpage中描述PoolSubpage分配内存的时候，allocateSubpage方法开头就会调用PoolArena：

或者在Netty之旅（三）ByteBuf内存管理中，PooledByteBufAllocator来分配堆内存：

为什么还是PoolArena？由于PoolSubpage的存在，整个Chunk被分成了更加碎片化的小内存，当分配大内存的时候，很可能会因为小内存的分配占用，导致无法找到连续的大段内存来进行分配。不过PoolArena的出现，这些问题都迎刃而解了。
  PoolArena的设计借鉴了jemalloc内存分配，下面是内部的骨架：

  1. tinySubpagePools和smallSubpagePools都是PoolSubpage结构，其中tinySubpagePools分配小于512字节的内存，smallSubpagePools用于分配大于等于512字节的内存（注意PoolSubpage小于一页）。
  2. PoolChunkList将PoolChunk按利用率拆分，这种形式的拆分极大保障了内存分配的成功率，下面是各个PoolChunkList的含义：
    • qInit：内存利用率0-25%的chunk链。
    • q000：内存利用率1-50%的chunk链。
    • q025：内存利用率25-75%的chunk链。
    • q050：内存利用率50-100%个chunk链。
    • q075：内存利用率75-100%个chunk链。
    • q100：内存利用率100%chunk链。

Thread Cache

  Thread Cache就一个类，PoolThreadCache，内部有三种分配方式，分别为tiny、small、normal：

tinyCache数组大小32，包含容量512的PoolChunk队列；smallCache数组大小4，包含容量256的PoolChunk队列；normalCache数组大小12，包含容量64的PoolChunk队列
