拥抱Docker（二）Dockerfile制作镜像

发表于 2019-06-04 | 分类于 Docker |

  在Docker的世界里有三个基本概念：

  • 镜像（Image）
  • 容器（Container）
  • 仓库（Repository）

  镜像就像一个打包好的应用程序，仓库则是应用程序的市场。我们去仓库下载镜像，然后在容器中运行这个镜像。容器的本质类似一个运行的进程，他是镜像的运行环境。Docker镜像极大地简化了应用的安装运行，我们不仅需要学会使用镜像运行容器，还需要学会如何去制作一款镜像，所以下面的任务是谈谈如何使用Dockerfile来制作一款镜像。

Dockerfile常用指令

Dockerfile的每一条指令都是对镜像做出一次修改，每次修改都会提交一个新的镜像层。

FROM

FROM是Dockerfile中第一条指令，它指定了一个基础镜像，后续的指令都是基于此镜像。上面nginx的例子基于alpine镜像，而alpine镜像基于Alpine Linux操作系统，Alpine Linux是轻量级的Linux发行版。每一个Dockerfile文件都必须继承于一个基础镜像，除了alpine这些操作系统镜像，Docker Hub上还有非常多高质量的官方镜像，如nginx、redis、mysql、tomcat等。除此之外，Docker还存在一个特殊的镜像scratch，这个镜像其实是一个虚拟的镜像，实际并不存在。如果你不想以任何镜像为基础，那么可以指定scratch作为基础镜像。

MAINTAINER

指定镜像作者

EXPOSE

声明运行时容器提供服务的端口，注意声明并不是指运行时就使用这个端口。在使用docker run -P运行容器的时候，会自动映射EXPOSE的端口。

RUN

最常用的指令之一，用于执行命令。之前说过，Dockerfile中每一条指令都会建立一层镜像，RUN指令也是如此，为了避免生成太多的镜像，一般用&&将各个命令联合起来，这样就不会生成多余的镜像。RUN指令有两种格式：

  • shell: RUN <命令>
  • exec: RUN [“可执行文件”, “参数1”, “参数2”]
CMD

容器启动的时候执行命令，同RUN一样有两种格式。

ENTRYPOINT

ENTRYPOINT和CMD类似，CMD命令有一个特点，它可以被docker run后面的命令参数替换掉，而ENTRYPOINT不会。

COPY

复制构建上下文目录（比如上面例子中的dockerFile_demo）下的文件或者文件夹到目标路径下，源路径可以有多个。

ADD

类似COPY但是拥有更多高级功能，官方推荐使用COPY，所以这里不多介绍。

ENV

设置环境变量，后面的指令或者运行中的应用都可以使用这个环境变量

VOLUME

在容器内存储数据，数据层会随着容器的删除而删除，Docker采用volume（卷）的形式来向容器提供持久化存储：

任何向/data中写入的数据都会写入一个匿名的数据卷中，即使容器被删除了，这些数据依然会保留。

WORKDIR

指定容器内的一个目录， 在容器内执行的命令会在该目录下执行。

ONBUILD

ONBUILD比较特殊，它后面会跟着其他的Dockerfile指令，比如RUN、CMD等，不过这些指令不会在当前镜像构建时立即执行，只有以当前镜像为基础镜像，去构建下一级镜像的时候才会被执行。
