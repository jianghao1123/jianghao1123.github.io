拥抱Docker（三）基础原理之Namespaces

发表于 2019-07-08 | 分类于 Docker |

  在学习了使用DockerFile制作一个简单的镜像并运行，我觉得接下去有必要了解一下Docker背后的运行原理，本着知其然而知其所以然，这样对后续学习或者使用Docker部署应用都有一定裨益。我们已经知道Docker是一个使用了虚拟化技术的容器，同虚拟机一样有着资源隔离和分配的优点，但是因为本身架构的不同，比虚拟机更加高效和便捷。既然如此，那么Docker是如何实现的？一切都还要从Linux的虚拟化技术说起。

Namespaces（命名空间）

  Namespaces是Linux Kernel的一个功能，它可以隔离一系列的系统资源，比如PID（进程）、Network（网络）、User ID(用户)、Mount（挂载）等等。Linux一共有六种类型的Namespaces：

类型 参数 描述
PID CLONE_NEWPID 进程ID（Linux 2.6.24）
Mount CLONE_NEWNS 磁盘挂载（Linux 2.4.19）
Network CLONE_NEWNET 网络（Linux 2.4.19）
IPC CLONE_NEWIPC 进程间通信，System V IPC和POSIX message queues（Linux 2.6.19）
UTS CLONE_NEWUTS hostname和domainname（Linux 2.6.19）
User CLONE_NEWUSER 用户组ID（Linux 3.8）

  上面这六种Namespaces主要由以下三种函数调用方式：

  1. clone() 创建新进程。
  2. unshare() 将进程移出某个Namespaces。
  3. setns() 将进程加入到某个Namespaces。

Mount Namespace

  先说一下什么叫Mount（挂载），在Linux中只有一个根目录，凡是根目录以外的文件想要被系统访问，就必须关联到这个根目录下的某个目录，这种关联动作就称之为“挂载”，而这个目录称之为“挂载点”。挂载的实质是为访问添加入口，挂载点目录在挂载之后有几个特性需要注意：

  1. 该目录下原有的文件会被隐藏。
  2. 该目录不可被其他进程使用。
