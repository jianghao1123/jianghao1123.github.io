    什么是软件 3.0

    本文是公司内部分享文档摘录。

    OpenAI 联合创始人、特斯拉前 AI 负责人 Andrej Karpathy 在 Y Combinator 的 AI 创业学院活动上，进行了个人演讲。

    本次演讲中，Karpathy 提出了「软件 3.0 时代」这一概念，他认为自然语言正在取代传统代码，而大型语言模型（LLM）则成为新的「万能计算机」。

    • 软件 1.0 就是我们熟悉的传统编程，程序员一行行写代码，告诉计算机该做什么。就像盖房子，每块砖头放在哪里都要亲自指挥。

    • 软件 2.0 就不一样了，核心变成了训练神经网络。你不用告诉计算机具体怎么做，而是给它大量数据让它自己”学会”。

    举个特斯拉的例子：最初的自动驾驶系统里有大量传统代码，比如遇到路障怎么办、看到行人怎么处理，这些都要程序员一条条写出来。但随着AI能力提升，神经网络不仅能识别汽车，连路上的牛、鸡、鸭、鹅都能认出来，那些传统代码就没用了，直接删掉。

    • 软件 3.0，就是现在正在发生的事情——我们可以用自然语言直接跟AI对话，让它执行任务。这比之前的神经网络更可控，也大大降低了软件开发的门槛。任何人都能用自然语言”编程”了。

    总结一下自己从视频中提炼的三点：

    • 现阶段 AI 产品的最好方式不是盲目追求全自动化的智能体，而是把传统的用户界面和大模型相结合，让 AI 去部分辅助人类进行工作。

    • 为 AI 开发内容，让 AI 更好地阅读并服务人类。未来互联网内容的形式，为 AI 智能体构建新的内容。AI 并不擅长解读 HTML，也不像人类，比如看到某个说明文档，点击按钮查看之类的操作，AI 并不擅长。

    • 程序员即将淘汰？Karpathy认为现在是进入软件行业的最佳时机。大量的软件都需要被改写和重新构思。以前的软件是为人类编写的，现在要考虑两点：

      • 智能体如何更方便地控制，以便服务人类。

      • 人类如何更方便地审查结果。

    尝试 Vibe Coding

    氛围编程 (vibe coding) 是一种新兴的软件开发方法，极大地依赖于人工智能 (AI)。该术语由 AI 研究员 Andrej Karpathy 于 2025 年初提出，用于描述一种流程，在该流程中，开发者主要使用自然语言提示（以普通语言说出或输入），来指示 AI 工具生成、优化和调试代码。

    使用 Cursor 开发

    在 cursor 的 agent 使用过程中，会产生如下问题：

    新兴的开发模式展望

    01 AI 原生 Git：为 AI 代理重新构想版本控制

    代码成为输入的副产品，更像是一个编译产物，而非人工撰写的源码。在这种世界里，Git 的功能会逐渐从协作工作区转变为一个产物日志 —— 一个不仅记录“什么改变了”，还记录“为什么改变”、“是谁操作的”的地方。

    code is cheap, show me the talk

    02 仪表盘->综合：动态AI驱动界面

    仪表盘的演变：如何更好地服务于人类用户与 AI 代理的双重需求

    03 文档，工具、索引和交互式知识库的结合体

    用户不再是从目录开始逐页阅读或自上而下浏览，而是从一个问题开始：“我该如何完成某项任务？”这种心理模型的转变，意味着文档的角色已经从“让我来学习这份规范”转变为“请帮我以我喜欢的方式重新组织这些信息”。

    文档不再仅仅是静态的 HTML 或 Markdown 页面，而是演变为由索引、嵌入（embeddings）和具备工具意识的AI代理支持的交互式知识系统。

    ▲ Mintlify的屏幕截图，用户可以使用cmd+k快捷键打开 AI 聊天窗口，对Mintlify 文档进行问答

    04 从静态模板到智能生成：Vibe编程正在取代Create React App的时代

    随着“文本生成应用”平台（如 Replit、Same.dev、Loveable、Convex 推出的 Chef 和 Bolt）以及 AI 集成开发环境（如 Cursor）的兴起，这种模式正在发生转变。

    05 异步代理工作的兴起

    异步代理编程的兴起，比如Google Jules、Manus 以及 Minimax Agent。

    06 基础能力模块化

    正如早期 Web 开发时代需要 Rails 的生成器和 rails new 来加速开发，AI代理时代也需要值得信赖的基础模块：即插即用的身份系统、使用追踪、计费逻辑和访问控制机制——这些模块不仅要足够抽象以便于代码生成，还要足够灵活，能够随着应用一起成长。
